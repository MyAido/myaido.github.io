Transparency Changelog

A complete history of permission changes, privacy updates, and transparency improvements.

Why This Matters

This changelog tracks every change to Aido's permissions, data handling, and transparency practices. If we ever add new permissions or change how data is processed, it will be documented here with a clear explanation.

Version 1.0.0 January 2026

Initial Release - Transparency Center Launch

Permissions Declared (7 total)

  • BIND_ACCESSIBILITY_SERVICE - For text reading and replacement
  • BIND_INPUT_METHOD - For custom keyboard functionality
  • INTERNET - For AI API calls
  • SYSTEM_ALERT_WINDOW - For overlay popups
  • ACCESS_NETWORK_STATE - For offline detection
  • FOREGROUND_SERVICE - To keep service active during use
  • REQUEST_IGNORE_BATTERY_OPTIMIZATIONS - For service stability (optional)

Privacy Features

  • Local-first architecture: No cloud storage of user prompts
  • App blacklist feature for banking/password apps
  • Encrypted SharedPreferences for API keys (AES-256)
  • Zero analytics or tracking SDKs

Transparency Documentation

  • Published complete Transparency Center with 11+ pages
  • Disclosed all third-party AI providers (Gemini, OpenAI, Groq)
  • Documented code behavior and logic flow
  • Established Zero Tolerance Policy
